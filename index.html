UX REDESIGN

APPSC Portal Redesign

Simplifying candidate onboarding and government recruitment application pipelines.

Problem Statement

Government recruitment portals are characterized by heavy information architectures, dense text grids, and circular application stages. Candidates often struggle with step-by-step documentation, complex registration flows, and hard-to-track exam schedules. This leads to user frustration, long completion times, and validation errors.

Research & Analysis

Heuristic Evaluation

Conducted an audit on the current interface finding violations of Aesthetics and Minimalist Design and Help Users Recognize/Recover from Errors. Forms had overlapping validation messages and lacked live progress tracking.

Competitor Analysis

Analyzed modern job application systems (LinkedIn, Swiggy Careers, tech job boards) to map expected user patterns for profiles, status dashboards, and upload validations.

Information Architecture & User Flows

Redesigned the application pipeline into three linear segments: Profile Creation (Document Repository), Job Filter, and Final Application Submission. By saving document configurations directly to the user account, candidate verification is automated, eliminating repetitive uploads.

User Flow Pathway

User Logs In → Accesses Unified Dashboard → Fills Profile Metadata → System Validates Documents → Simple 1-Click Application to Active recruitments → Instant Confirmation.

Design System & Accessibility

Typography & Colors

Selected highly readable fonts and created a clean primary hierarchy utilizing deep blue branding for trust, with high-contrast text tags (WCAG AAA compliant contrast ratios for legibility).

Accessibility (WCAG)

Implemented keyboard-navigable tabs, explicit aria labels on dynamic forms, and strong color differentiators for warning/success validation states.

AI-Assisted Exploration

Leveraged mid-journey AI UI layouts and prompt-based exploration to rapidly experiment with component micro-animations and cards placement, streamlining iterations before finalizing high-fidelity mockups in Figma.

Outcome & Impact

50%

Reduction in form submission errors by introducing real-time field validation and structured progress bars.

1-Click

Repeat application flow enabled by storing user profile validation data securely.

MOBILE DESIGN

Code Companion

Mobile-first programming learning assistant built on gamification and progress mapping.

Problem Statement

Students starting out in programming face learning fatigue due to abstract syntax concepts, disjointed tutorial portals, and a lack of daily reinforcement prompts. Desktop-only environments limit when and where they can practice basic syntax.

User Research & Personas

Surveyed 50+ computer science freshmen. Key finding: 78% of students abandoned online coding courses due to long, dry video lectures. They preferred interactive quiz modules, clear visual journey tracking, and compiler sandboxes that they could test on mobile during daily commutes.

Persona: Rahul (The Novice)

Needs: Short bites of theory followed by instant interactive puzzles. Needs streaks to stay motivated.

Persona: Ananya (The Practical Learner)

Needs: Clean cheatsheets, quick syntax references, and a simple feedback loop to check code errors.

Information Architecture & Wireframing

Configured a bottom navigation model for standard thumb range accessibility:

  • Home Dashboard (Staged progress track, daily streak meter)
  • Learn Module (Interactive code cards and single-answer puzzles)
  • Practice Sandbox (Simplified code editor with pre-filled syntax templates)
  • Profile (Certificates, leaderboard rankings)

Prototyping & Usability Testing

Built high-fidelity mobile prototypes in Figma. Ran task-based usability testing with 5 users. Iterated on the compiler interface: simplified output displays and enlarged tap targets for key coding symbols (e.g., curly brackets, semicolons) on mobile keyboards.

Outcome & Impact

94%

User satisfaction score for the mobile-optimized coding keyboard adjustments.

3x

Increase in daily self-practice frequency compared to web-only portals.

DATA EXPERIENCE DESIGN

Renewable Energy Forecast

Designing an information dashboard to simplify global environmental trend projections.

The Design Challenge

Decision-makers (policymakers, environmental researchers) have to sift through multi-page energy tables to compare regional production statistics and forecast models. Dense data formats result in cognitive overload, making it difficult to spot positive transitions or predict local energy shortfalls.

Dashboard Experience & Information Design

Designed a visual exploratory system that centers on user intent:

  • Global Heatmap View: Allows quick spatial analysis of energy production (Solar, Wind, Hydro).
  • Dual-Axis Comparative Charting: Compares target thresholds against projected yields till 2030.
  • Interactive Query Drawer: A sliding control panel that lets users refine regions without losing their current map coordinates.

User-Centered Data Exploration

Applied progressive disclosure: users start with macro KPIs (total global yield, year-over-year growth) and can drill down into regional lines. Replaced cluttered tables with intuitive visual cards containing trend sparklines and status indicator tags (e.g., "On Track", "Needs Intervention").

Outcome & Experience Impact

90%

Reduction in manual data sorting times by replacing long, static documents with a visual dashboard.

Clear IA

Enabled non-technical stakeholders to identify regional anomalies under 10 seconds.

INFORMATION DESIGN

EDA of Indian Demographics

Translating complex population spreadsheets into digestible visual narratives.

The Design Challenge

National Census surveys contain complex socioeconomic metrics (literacy, gender ratios, regional growth rates, urban density). Because this data is buried in massive grids, researchers and local journalists find it difficult to correlate variables or identify disparities across districts.

Data Storytelling & Layout

Structured the analytics as a visual narrative divided into thematic sections:

  1. The Big Picture: High-level cards summarizing total metrics.
  2. Regional Disparities: Interactive bubble map highlighting literacy variations.
  3. Correlation Analysis: Scatter plots indicating how gender ratios relate to regional education indicators.

Information Architecture & UX Audit

Created a visual hierarchy by establishing consistent typography standards, high-contrast grid lines, and responsive layouts. Replaced dense legend blocks with interactive tooltips that display descriptive tooltips on hover, keeping the main interface clean and focused.

Outcome & Narrative Impact

Clean UX

Simplifies exploration of demographic anomalies by linking maps and charts dynamically.

Zero Code

Interactive tooltips and visual legends allow non-technical audiences to extract insights instantly.
